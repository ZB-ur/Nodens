## Goal
构建一个自部署的小红书矩阵运营工具，通过 LLM 自动生成文案 + Playwright 浏览器自动化实现多账号定时批量发布，解放自媒体运营人力。

## Features
- F-001 account-management: 矩阵账号管理 — 支持通过扫码或导入 Cookie 添加 1-5 个小红书账号，加密存储登录态，定时心跳检测 Cookie 有效性（每30分钟），失效时通过 Dashboard 通知用户重新扫码登录
- F-002 topic-config: 主题方向配置 — 每个账号可独立配置内容主题方向（如美妆、穿搭、美食），支持自定义关键词、风格描述，作为 LLM 文案生成的输入参数
- F-003 content-generation: LLM 文案生成 — 通过 Claude CLI 根据账号主题配置自动生成小红书风格笔记文案（标题≤20字 + 正文≤1000字 + 话题标签），内置小红书风格 Prompt（emoji、口语化、种草语气），支持用户自定义 Prompt 模板，生成后支持预览和手动编辑
- F-004 image-library: 图片素材库管理 — 用户手动上传图片到素材库（支持 JPG/PNG/WEBP，单张≤10MB），按账号/主题分类管理，发布时从素材库选取配图（1-9张），支持拖拽排序
- F-005 schedule-engine: 批量排期系统 — 用户设定发布规则（每天篇数、发布时段），系统自动生成一周发布排期表，内置推荐发布时段（早10点、晚8点等流量高峰），默认每账号每天≤2篇、发布间隔≥2小时，支持日历视图查看和手动调整排期
- F-006 auto-publish: 浏览器自动化发布 — 通过 Playwright 在排定时间自动登录小红书并发布图文笔记，内置反检测措施（随机延迟200-2000ms、鼠标轨迹模拟、stealth plugin指纹伪装、操作频率控制），发布失败时自动截图保存现场，选择器配置化（不硬编码于代码中）以应对 DOM 变化
- F-007 publish-monitor: 发布监控 Dashboard — 展示各账号发布状态（待发布/发布中/成功/失败）、成功率统计、排期日历视图、失败原因和截图查看、Cookie 状态告警，支持手动重试失败任务
- F-008 anti-risk-config: 安全策略配置 — 提供全局和账号级安全参数配置（每日发布上限、最小发布间隔、随机延迟范围），默认保守策略，产品内明确提示浏览器自动化的合规风险
- F-009 selector-config: 选择器热更新 — 小红书页面 DOM 选择器（登录、发布等关键流程）配置化存储于 JSON/YAML 文件中，支持通过管理后台修改选择器而无需改代码，预留版本管理

## Constraints
- 小红书无官方发布 API，所有发布操作必须通过 Playwright 浏览器自动化模拟用户操作
- 账号登录态依赖 Cookie，Cookie 有效期不可控，需实现过期检测 + 通知重新登录的闭环
- LLM 调用通过 Claude CLI（`claude -p`）实现
- 前端技术栈：React + TypeScript + Tailwind CSS
- 后端技术栈：Node.js + Express/Koa
- 数据库：SQLite（自部署友好，无需额外数据库服务）
- 需要后台常驻进程（node-cron 或 bull queue）处理定时任务调度
- 部署环境需支持 headless Chromium（Playwright 依赖），Docker 部署需特殊配置
- Cookie 等敏感信息必须加密存储
- 浏览器自动化操作处于平台 ToS 灰色地带，产品需明确合规风险提示
- 默认反风控策略：每账号每天≤2篇，发布间隔≥2小时，随机延迟200-2000ms
- 架构需预留平台适配器抽象层，便于后续扩展知乎等平台

## Out of Scope
- 知乎、微博等其他平台支持（MVP 仅支持小红书）
- 视频笔记发布（MVP 仅支持图文笔记）
- AI 自动生图（图片需用户手动上传）
- 数据分析与粉丝管理功能
- 评论自动回复
- A/B 测试（文案/发布时段对比测试）
- 多用户协作与权限管理（MVP 为单用户系统）
- 违禁词/敏感词自动检测
- 爆文复刻/竞品分析功能
- 移动端适配（MVP 仅 Web 端）
